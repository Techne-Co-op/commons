Schema Reference Card

The 31 tables, their primitives, their anchors, and exactly what is committed
C6 · VERSION 0.1 · DRAFT FOR REVIEW · JUNE 2026
REGENHUB, LCA · PREPARED BY NOU · consumed at W0.7, W1.x, W2.1 · repo home: docs/schema-card.md
CIS PRD v0.2 §06·core tables and primitives Hub PRD v0.1 §04·extension tables plan v0.1 + CONVENTIONS·envelope and constraints

A number without provenance is a rumor, and so is a column. This card consolidates every table the PRDs name, and tags each fact with its source so an agent writing migrations never mistakes a proposal for a commitment.

01 · How to Read This

Each card carries three kinds of line, named by source. A prd line restates what a PRD commits: the table's existence, its REA primitive, its governing anchor, and any behavior the PRD text states outright. A plan line restates what the implementation plan and docs/CONVENTIONS.md commit: the event envelope, constraints, and conventions already adopted in the seed. A line marked proposed is mine: the minimum identity and reference shape a migration needs, offered for adoption and binding on no one. The discipline matters because this card feeds W0.7 and every schema step after it; an agent may build from prd and plan lines directly, and must treat proposed lines as a draft to confirm in the PR.

What this card is not: a migration, a column-complete data dictionary, or a substitute for counsel on any field that touches tax, securities, or member rights. Where the PRDs are silent, this card stays close to silent, and section 06 lists the silences worth deciding rather than discovering.

02 · Shared Shape

Two structural commitments apply across the whole schema before any single table is interesting. First, the REA discipline: every table traces to one of the four primitives, and every relationship between tables is an Event (CIS §06). Second, the event envelope from docs/CONVENTIONS.md: every state change is an event row, and tables of kind Event share the envelope columns rather than inventing their own.

event envelope · shared columnsplan CONVENTIONS · CIS principle 09

The audit posture in column form. Mutations on enveloped tables write an audit row via the generic trigger; pgTAP asserts prior-value capture.

committedplanid uuid pk · actor uuid not null · occurred_at timestamptz not null default now() · entity text · entity_id uuid · prior_value jsonb · payload jsonb · citation text not null
identity and reference shapeproposed for adoption at W0.7

The minimum every card below assumes when a proposed line names a reference.

proposeduuid primary keys throughout · created_at timestamptz default now() on entity tables · foreign keys written as agent_id -> agents · REA primitive and anchor recorded as a table comment per CONVENTIONS

cards below omit envelope columns on Event tables and the proposed identity shape; both are implied

06 · Silences Worth Deciding

Five places the PRDs name a behavior but no table, or a table but no shape, surfaced by the act of consolidating. None blocks W0.7; each should be decided at the step that first touches it rather than discovered inside a migration.

First, the Conflict & Disclosure module (CIS §08) has no table among the nineteen: director disclosures, recusal tracking, and dispute state need a home, most likely a disclosures table proposed at the W4 boundary. Second, the lifecycle state machine (CIS §08, Phase 04) names states and clocks but no storage; whether status lives on memberships with transitions as events, or in a dedicated table, is a W5.0 authoring question. Third, tax identifier encryption is committed but the mechanism (pgcrypto column encryption versus application-layer envelope) is not; it touches the accountant's workflow and belongs beside W3.5 with a security review. Fourth, notices §X channels: the multi-channel commitment is named while the channel set beyond email is not; schema should hold channel as open text until D-level work names the second rail. Fifth, Preferred Stock fields beyond existence are reserved with the investor framework (CIS §17); the card deliberately proposes none.
