## Question

Which tool builds the member portal.

## Context and options

PRD criteria: portability, accessibility, mobile-first, Supabase compatibility, design-system fidelity, plus the chokepoint test (CIS §17). Proposed default: a static-first framework with islands of interactivity (Astro is the type specimen; Vite + React SPA is the alternative).

## Proposed default

Stated in the context above; a proposal only. The decision is reserved to the named owner.

## Record of decision

Decided 2026-06-12 by Todd Youngblood (Ventures & Operations Steward).

Astro. Static-first with islands of interactivity — matches the existing commons design language (vanilla HTML, CSS token layer), ships zero JS for public pages, adds auth-gated interactive components as islands for the member portal. Meets all PRD criteria: portable flat-file output, strong accessibility baseline, mobile-first by default, Supabase JS client works in island components, and passes the chokepoint test — no framework lock that prevents escape. Revisit if Astro's island model proves insufficient for portal complexity, but the default stands until that bar is met.
