## Question

How the Ethereum Everywhere liaison reviews KPIs.

## Context and options

CROPS implications cut both ways (Hub §11). Proposed default: scoped read-only dashboard credential, with the public KPI view satisfying the open-source pull.

## Proposed default

Stated in the context above; a proposal only. The decision is reserved to the named owner.

## Record of decision

Decided 2026-06-12 by Todd Youngblood (Ventures & Operations Steward).

The public KPI surface is the EE liaison interface. D6 establishes an open data stance: hub metrics, event attendance, and KPI aggregates are publicly readable via the anon key with RLS, requiring no credential. The EE liaison accesses the same public page as any member of the public — a URL, not a login. No scoped read-only credential is provisioned. If the EE team requires reporting formats beyond what the public page provides (e.g. CSV export, digest email), that is handled as a reporting feature request, not an access control decision. The CROPS boundary is respected: the public surface exposes aggregates only, never individual member data.
