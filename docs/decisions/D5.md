## Question

Whether Make.com is ever adopted, and what would justify it.

## Context and options

Proposed default: not adopted in alpha. Trigger if invoked: a required integration whose first-party Edge Function cost exceeds reasonable effort and whose data remains exportable. Until then its PRD duties route to pg_cron, Edge Functions, and Actions.

## Proposed default

Stated in the context above; a proposal only. The decision is reserved to the named owner.

## Record of decision

Decided 2026-06-12 by Todd Youngblood (Ventures & Operations Steward).

Make.com is not adopted. The principle is use-what-we-have-first: GitHub Actions and Supabase Edge Functions are already in the stack and cover all automation duties assigned to Make.com in the PRDs. No additional third-party tool is adopted until existing access is exhausted. If Make.com is ever reconsidered, the bar is a required integration whose first-party implementation cost is unreasonable and whose data remains fully exportable — both conditions must hold.
