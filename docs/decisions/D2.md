## Question

Where the public application form lives.

## Context and options

The PRDs frame Formbricks (self-host) versus Tally or Typeform (managed chokepoint) (Hub §11). The constraint adds a third option, named explicitly as an extension: a static form on Pages inserting through a narrowly scoped anonymous RLS policy. Proposed default: the native path, Formbricks as fallback.

## Proposed default

Stated in the context above; a proposal only. The decision is reserved to the named owner.

## Record of decision

Decided 2026-06-12 by Todd Youngblood (Ventures & Operations Steward).

Native GitHub Pages form with a narrowly scoped anonymous Supabase RLS insert policy. The form lives as a static HTML page in the Pages deployment; on submission it writes directly to an applications table via the anon key, with RLS restricted to insert-only on that table — no read, no update, no delete. No third-party form host is adopted. This is consistent with D5 (use what we have), D6 (Supabase auth as the access layer), and the chokepoint test. Formbricks and managed alternatives are not adopted.
