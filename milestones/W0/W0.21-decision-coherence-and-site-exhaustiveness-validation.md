## What this step does

Extends `scripts/validate.py` with three new checks:

1. **Decision-coherence check.** For every file in `docs/decisions/` with a
   valid `id` field: the matching ledger entry in `milestones/D/` or
   `milestones/R/` exists; the `ledger_entry` path field resolves; the status
   pair is coherent (`open` ↔ `pending`/`blocked`; `decided` ↔
   `done`/`verified`). The exact class of drift that hit the public page — a
   badge reading "resolved" while the brief says "open" — becomes a CI failure.

2. **Site-exhaustiveness widening.** The current exhaustiveness grep fires only
   on paths matching `^(supabase|apps|design|scripts)/`. This extension adds
   `site/` to that pattern. Public-surface changes now require a milestone touch
   like everything else. The uncovered path identified in the drift inventory
   is closed.

3. **Manifest-link check.** (Inside `build_site.py` or the validator.) Every
   `manifest.yaml` entry's file exists; every library file appears in the
   manifest; internal links between documents resolve. Link rot becomes visible
   at PR time rather than after deploy.

## Why this runs parallel to W0.19

The decision-coherence check and exhaustiveness widening are independent of the
library restructure — they operate on the current `docs/decisions/` and whatever
path prefix the site uses. W0.21 can land as soon as W0.2 is done, without
waiting for W0.19 to complete.
