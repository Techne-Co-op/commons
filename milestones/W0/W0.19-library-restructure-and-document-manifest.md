## What this step does

Moves the eleven HTML documents from the repository root into `site/library/`,
each in its own subdirectory with the version in the filename and a stable
`index.html` pointing at latest. Creates `site/library/manifest.yaml` as the
machine-readable document register.

## Why

Every link to `cis-prd-v0.2.html` dies when v0.3 ships. Under this scheme
`…/library/cis-prd/` is a permanent address; versions remain addressable
beside it. The manifest makes machine-readable what the Builder's Collection
Map already says in prose, ending the "five primary documents" class of staleness.

## manifest.yaml structure

Each entry carries: `id`, `title`, `role` (specification / plan / instrument /
collection), `version`, `date`, and `consumed_by` (list of W-step ids). The
document library section of the front page and the collection map's index both
render from this manifest.
